小红书「收藏夹」功能体验优化

解决"收藏即吃灰"核心痛点 —— 以「轻管理、重回顾」为策略，通过智能标签归类、收藏周报推送、沉浸式回顾模式三项功能，提升收藏内容利用率与用户回访意愿。

📅 2026.05 ｜ 独立完成 📱 iOS / Android
01

项目概述

📌 背景

小红书用户日均产生大量收藏行为，但绝大多数收藏内容在保存后便不再被打开，形成"收藏即吃灰"现象。大量内容沉淀在收藏夹中，缺乏有效的组织管理和二次触达机制，收藏功能的用户价值远未被释放。

🎯 目标
  • 收藏 7 日回访率提升 30%+（核心指标）
  • 收藏二次消费时长提升至 15s+
  • 降低用户收藏管理负担，提升收藏内容利用率
📐 范围
  • 覆盖端：iOS / Android 移动端
  • 涉及模块：收藏夹首页、收藏流程、消息推送、个人页
  • 不涉及：推荐算法改造、首页信息流
02

市场与竞品分析

选取抖音、Pinterest、得到三个维度不同的产品进行对标

维度 小红书（现状） 抖音 Pinterest 得到
收藏分类方式 手动建收藏夹 无分类，全部平铺 画板（Board）体系 文件夹 + 标签
智能归类 ❌ 无 ❌ 无 ✅ AI 自动分类建议 ✅ 智能标签
二次触达机制 ❌ 无 ⚠️ 收藏列表入口浅 ✅ 每周精选推送邮件 ✅ 学习提醒
回顾体验 列表平铺 列表平铺 瀑布流 + 相似推荐 知识卡片回顾
可借鉴点 - 收藏即订阅的轻量感 视觉驱动回顾、每周推送 标签管理体系
核心洞察

抖音验证了"轻分类"路线的可行性——在用户量极大的场景下，弱化管理反而提升了收藏使用率。Pinterest 的每周精选邮件是"重回顾"方向最成熟的方案，已被验证可显著提升回访率。得到的标签体系可轻量化借鉴，但过于重度的文件夹管理不适合小红书用户习惯。

03

用户研究

通过用户访谈与行为数据分析，归纳出三类典型用户

TYPE A · 重度收藏党

收藏即弃

📦 500+ 收藏 📅 每日 5-10 条 🏷 从不分类

每天收藏大量内容，但从不整理。想回顾时需要翻找很久，越积越多就彻底放弃了整理的念头。收藏夹对他来说就是"内容的黑洞"。

"我也知道该整理，但每次收藏完就忘了，越堆越多就更不想动了"
TYPE B · 轻度假日党

收藏即已读

📦 50- 收藏 📅 月均 15 条 🏷 几乎不回看

收藏行为多是"先存着再说"，很少主动回去翻看。收藏对他来说是心理安慰而非实际需求，缺乏回访的动机和触发点。

"收藏就等于我看了，之后不会专门点进去翻"
TYPE C · 整理控

挣扎的分类者

📦 200+ 收藏 📅 每周整理 🏷 坚持但吃力

有强烈的整理习惯，每条收藏都会认真分类。但手动整理耗时费力，经常拖一两周就放弃一次，然后陷入"整理-放弃-堆积-再整理"的循环。

"整理太累了，有时候拖两周就放弃了，然后看着一堆未分类的更难受"
用户类型 痛点场景 需求描述
重度收藏党 旅行前想起收藏过攻略，打开收藏夹翻了 5 分钟找不到，最后直接去搜索了 系统自动归类，不需要手动整理也能快速找到想要的内容
轻度假日党 周末无聊想看看收藏了啥，点进去发现全是几个月前的，已经没兴趣了 系统在合适的时候主动提醒，创造"再看一眼"的契机
整理控 每周日晚上花 30 分钟整理收藏，终于坚持了 3 周后放弃了 系统做大部分归类工作，用户只需微调确认

05

功能需求详解

🏷️

功能一：智能标签归类

▼
📋 功能概述

用户收藏笔记时，系统基于 NLP 内容识别实时分配标签，替代现有手动创建/选择收藏夹的流程。用标签体系彻底替换收藏夹体系。

⚙️ 业务逻辑
  • 用户点击收藏按钮 → 系统实时识别内容主题 → 分配 1-2 个标签 → 收藏成功 Toast 中直接展示标签，右侧提供"编辑"入口
  • 识别置信度 ≥ 60% 时自动分配标签；< 60% 时标记为"未分类"，Toast 提示"暂未识别到内容类型，点击手动选择"
  • 收藏夹首页按标签 类目折叠分组 展示（如"生活"类目下含穿搭、美食、家居标签），默认展开最近使用的前 2 个类目，其余折叠
  • 标签管理页支持合并、重命名、删除标签；单标签下内容超过 200 条时触发系统建议细分
🖱️ 交互细节
  • 收藏 Toast：底部弹出，文案"已收藏到 [标签名]"，右侧"编辑标签"按钮
  • 收藏夹首页顶部为标签类目折叠区，下方为内容列表（按当前选中标签筛选）
  • 支持"全部内容"视图（按时间倒叙）和"按标签浏览"视图
⚠️ 异常流程
  • 内容已被原作者删除 → 标签保留，内容显示"已失效"
  • 网络异常导致识别失败 → 暂标为"未分类"，后台重试识别
📬

功能二：收藏周报推送

▼
📋 功能概述

定期向用户推送收藏内容回顾报告，展示本周收藏概览、精选内容和待回顾提醒，用轻量化的信息流方式引导用户重新回到收藏内容。

⚙️ 业务逻辑
  • 默认推送时间：每周日 20:00
  • 支持用户自选推送频率：每 3 天 / 每周（默认）/ 每两周
  • 点击推送通知 → 进入 H5 收藏周报页面
📄 周报内容结构
  • 本周收藏概览：本周收藏数、已回顾数、累计总收藏数
  • 本周精选：本周收藏中热度 / 互动最高的 3 条内容
  • 待回顾提醒：展示 N 天前收藏但尚未回看的内容，可标记"稍后回顾"
⚙️ 设置项
  • 收藏夹设置页新增"收藏周报"开关
  • 支持频率选择（每 3 天 / 每周 / 每两周）
  • 推送通知可单独开关
  • 推送底部提供"管理推送偏好"入口
⚠️ 异常流程
  • 本周未收藏 → 不推送周报，改为推送"猜你喜欢"（基于历史收藏推荐）
  • 连续 4 周未点击周报 → 自动降低推送优先级或暂停推送
  • 用户关闭推送通知 → 周报仅在 App 内消息中心展示
🖼️

功能三：沉浸式回顾模式

▼
📋 功能概述

在收藏夹中新增"回顾"独立 Tab，进入后以全屏沉浸式卡片流展示收藏内容，用户上下滑动浏览，操作体验与首页刷笔记一致，降低回访心理门槛。

⚙️ 业务逻辑
  • 入口：收藏夹底部 Tab 栏新增"回顾"独立 tab
  • 默认排序：随机顺序（制造惊喜感），支持切换为"按标签筛选后随机"
  • 退出后记忆浏览位置，下次进入从中断处继续
🖱️ 手势操作
  • 上滑 / 点击：下一条
  • 下滑：上一条
  • 左滑：移除（取消收藏）
  • 右滑：标记"已回顾"
  • 点击内容区：跳转查看原始笔记详情
🖱️ 交互细节
  • 进入回顾模式时有平滑动效过渡
  • 全屏深色背景，只展示当前卡片内容
  • 顶部显示进度（如 3/27）、退出按钮
  • 底部有轻量手势操作提示
⚠️ 异常流程
  • 内容已被原作者删除 → 占位图 + "内容已消失"提示，自动跳过
  • 收藏内容不足 5 条 → 展示"快去收藏你喜欢的内容吧"空状态引导
  • 回顾中网络中断 → 已加载的缓存内容继续可滑动，加载失败项标注
06

数据指标体系

核心指标 + 过程指标，覆盖结果度量与过程监控

指标 类型 定义 当前值 目标值
收藏 7 日回访率 核心 收藏后 7 天内再次访问该收藏的用户占比 ~20% >26%（+30%）
收藏二次消费时长 核心 每次回访收藏内容的平均浏览时长 ~8s >15s
周报点击率 过程 收到周报推送后点击进入的用户占比 - >15%
回顾模式完成率 过程 进入回顾后浏览 ≥ 5 条的用户占比 - >60%
标签覆盖率 过程 已自动分类内容占总收藏数比例 - >85%
收藏功能 NPS 体验 收藏功能净推荐值 基线待测 +10pp
